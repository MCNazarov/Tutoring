# Лабораторная работа 2: Настройка мониторинга

## Цель
Настроить систему мониторинга для отслеживания состояния инфраструктуры, установив Prometheus в Docker на своем локальном компьютере.

## Задачи
1. Установить и настроить Docker на своем локальном компьютере.
2. Запустить Prometheus в Docker для сбора метрик с удаленных серверов.
3. Установить Grafana и настроить дашборды для визуализации собранных данных.

## Описание
В этой лабораторной работе студенты будут устанавливать Prometheus в Docker на своих локальных компьютерах и настраивать его для сбора метрик с удаленных серверов, таких как Windows и Ubuntu. Grafana будет использоваться для визуализации собранных данных. Студенты научатся создавать дашборды, отображающие состояние системы и производительность сервисов.

## Ресурсы
-     Docker
-     Prometheus
-     Grafana
-     SSH доступ к серверам
-     Файл инвентаризации Ansible из первой лабораторной работы
-     Файл переменных `vars.yml`, содержащий логины и пароли для доступа к серверам

## Подготовка
1. Убедитесь, что все серверы доступны и настроены.
2. Убедитесь, что файлы `inventory.yml` и `vars.yml` правильно настроены и содержат актуальные данные.
3. Установите Docker на свой локальный компьютер, если он еще не установлен. Инструкции можно найти на [официальном сайте Docker](https://docs.docker.com/get-docker/).
